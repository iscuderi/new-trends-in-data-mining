4. Modelado

En primer lugar se deben crear dos subsets, uno de training y otro de testing, para entrenar y testear los modelos. Asimismo, se aplica una semilla al inicio, de manera de hacer los resultados replicables.

Dividimos los datos de la siguiente manera:

  • 75% train
  • 25% test
set.seed(2021)
muestra.train=sample(1:dim(df)[1],round(dim(df)[1]*0.75),replace=FALSE)
df.train=df[muestra.train,]
muestra.test=setdiff(1:dim(df)[1],muestra.train)
df.test=df[muestra.test,]

A continuacin aplicamos los siguientes modelos de clasificacin:

-Logistic Regression

-Quadratic Discriminant Analysis (QDA)

-Naive Bayes

-rboles

Realizamos una breve descripcin de cada modelo:

Logistic Regression: Este mtodo permite estimar la probabilidad de una variable cualitativa binaria en funcin de una variable cuantitativa. La regresin logstica permite calcular la probabilidad de que la variable dependiente pertenezca a cada una de las dos categoras en funcin del valor que adquiera la variable independiente.

Quadratic Discriminant Analysis (QDA): Esta metodologa es similar a LDA, con la diferencia de que el QDA considera que cada clase k tiene su propia matriz de covarianza y, como consecuencia, la funcin discriminante toma forma cuadrtica.

Naive Bayes: El algoritmo Bayes naive es un algoritmo de clasificacin de variables cualitativas basado en los teoremas de Bayes. Este algoritmo es llamado ingenuo porque calcula las probabilidades condicionales por separado, como si fueran independientes una de otra. Una vez que se obtienen las probabilidades condicionales por separado, se calcula la probabilidad conjunta de todas ellas, mediante un producto, para determinar la probabilidad de que pertenezca a la categora. Luego se itera dicho proceso para cada observacin.

rboles: Mediante este modelo se intenta predecir una variable dependiente a partir de variables independientes. Existen rboles de clasificacin (variable discreta) y rboles de regresin (variable continua). Lo que hace este algoritmo es encontrar la variable independiente que mejor separa nuestros datos en grupos, que corresponden con las categoras de la variable objetivo. Esta mejor separacin es expresada con una regla. A cada regla corresponde un nodo.

Una vez hecho esto, los datos son separados (particionados) en grupos a partir de la regla obtenida. Despus, para cada uno de los grupos resultantes, se repite el mismo proceso. Se busca la variable que mejor separa los datos en grupos, se obtiene una regla, y se separan los datos. Hacemos esto de manera recursiva hasta que nos es imposible obtener una mejor separacin. Cuando esto ocurre, el algoritmo se detiene. Cuando un grupo no puede ser partido mejor, se le llama nodo terminal u hoja.

Las mtricas que utilizaremos para evaluar el rendimiento de los modelos son las siguientes:

Recall (TPR):

\[ \text {Recall = Verdaderos positivos / (Verdaderos positivos + Falsos negativos) } \] Es una forma de analizar la tasa de verdaderos positivos. Si su valor es bajo, quiere decir que hay muchos falsos negativos, lo que implica que muchas transacciones fraudulentas no fueron detectadas. Esta es la principal mtrica dentro de nuestro estudio.

Precision:

\[ \text {Precision = Verdaderos positivos / (Verdaderos positivos + Falsos positivos) } \] Si se obtiene un valor reducido, querr decir que hay muchos falsos positivos, es decir que hemos levantado sospechas sobre transacciones no fraudulentas. Si bien se desean evitar estas situaciones ya que posee un costo econmico (el banco deber comunicarse con el cliente para corroborar la operacin y realizar un anlisis en profundidad de la misma), es de menor gravedad que el caso planteado en el apartado anterior.

F:

\[ \text {Prueba F = (2 * Precision * Recall) / (Recall + Precision) } \] Es la media armnica de las anteriores mtricas.

Mtodos a aplicar:

Oversampling: Las tcnicas de oversampling buscan generar nuevos valores de la clase minoritaria a la hora de entrenar los modelos, de manera tal que estos sean capaces de predecir mejor en datasets desbalanceados como el nuestro. En nuestro caso utilizaremos el mtodo Smote, creando 2, 6, 15, 50 y 80 datos nuevos por cada uno ya existente de la clase minoritaria.

Undesampling: Las tcnicas de undersampling buscan eliminar datos de la clase mayoritaria a la hora de entrenar el modelo. En nuestro caso utilizaremos la funcin tomek link para dicho fin. Dicha funcin, busca los valores de la clase mayoritaria ms cercanos a los de la clase minoritaria y los elimina.

Funcin de coste: En la clasificacin regular, el objetivo es minimizar la tasa de clasificacin errnea y, por lo tanto, todos los tipos de errores se consideran igualmente graves. En cambio, en la clasificacin sensible al costo, los errores no se suponen iguales y el objetivo es minimizar el costo esperado.

En nuestro caso supondremos que el mayor costo est vinculado a no detectar operaciones fraudulentas (C = 5), seguido de predecir fraudes cuando no los hay (C = 1). Por ltimo, no se le ha asignado ningun costo a predecir correctamente para simplificar.

Se procede luego a analizar si V12 es mayor o igual a -2.2. En caso de ser as, la etiqueta a aplicar es la de la clase mayoritaria. En este subgrupo hay un 3% de los datos que quedaran mal etiquetados, ya que pertenecen a la clase minoritaria. Si en cambio el valor de V12 es menor a -2.2, la etiqueta es la correspondiente a la clase minoritaria (81% de los datos son de la clase minoritaria).
